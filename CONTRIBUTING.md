# Contributing to OpenCap Offline

First off, thank you for considering contributing to OpenCap Offline! This tool is built to make biomechanical analysis more accessible, and community contributions are highly valued.

There are several ways you can contribute to this project:

## 1. Reporting Bugs
If you run into an installation error, a crash during processing, or an unexpected bug in the GUI, please open an Issue on our [GitHub tracker](https://github.com/driscollh/opencap-offline/issues). 
When reporting a bug, please include:
* Your operating system (e.g., Windows 11).
* Your GPU model and driver version.
* A clear description of the problem.
* The exact error output from the terminal, if applicable.

## 2. Suggesting Enhancements
If you have ideas for new features, GUI improvements, or support for additional camera types, please open an Issue and tag it as an "enhancement." Describe how the feature would work and why it would be useful to the biomechanics community.

## 3. Contributing Code (Pull Requests)
If you want to contribute code to fix a bug or add a feature, please follow these steps:
1. **Fork** the repository.
2. **Create a branch** for your feature or bug fix (`git checkout -b feature/ExampleFeature`).
3. **Commit** your changes with clear, descriptive messages.
4. **Push** to the branch (`git push origin feature/ExampleFeature`).
5. **Open a Pull Request** describing the changes you have made.

We will review your pull request, test it against the existing offline pipeline, and merge it if it aligns with the project's goals.

## Code of Conduct
By participating in this project, you agree to maintain a respectful and collaborative environment for all researchers and developers.
